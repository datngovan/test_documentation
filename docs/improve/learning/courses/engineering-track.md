# Engineering Learning Track

> **Owner:** People Operations + Engineering Leadership | **Last Updated:** 2026-03-01

## Overview

The Engineering Learning Track defines structured learning paths for each career stage. All engineers receive **$2,500/year** learning budget and **2 hours/week** protected learning time (Fridays 2–4PM).

---

## Track 1: Junior Engineer (IC1–IC2)

**Goal:** Build foundational engineering skills and ramp to independent delivery within 6 months.

### Required Courses

| Course | Platform | Time | Completion Deadline |
|--------|----------|------|---------------------|
| Go Programming — Complete Guide | O'Reilly | 12h | Month 1 |
| PostgreSQL for Application Developers | O'Reilly | 8h | Month 2 |
| Docker & Kubernetes Fundamentals | Pluralsight | 10h | Month 2 |
| Git Deep Dive | O'Reilly | 4h | Month 1 |
| Writing Effective Tests in Go | Internal (Notion) | 2h | Month 1 |

### Recommended Projects (choose 1 per quarter)

- Implement a new step handler type (good first contribution to engine)
- Write missing test coverage for an existing module (improve coverage score)
- Add a dashboard panel to our Grafana setup for a metric that doesn't exist yet

---

## Track 2: Mid-Level Engineer (IC2–IC3)

**Goal:** Develop independent feature ownership, cross-service thinking, and mentorship skills.

### Required Courses

| Course | Platform | Time | Notes |
|--------|----------|------|-------|
| Distributed Systems Fundamentals | O'Reilly (Kleppmann) | 20h | Key text: DDIA chapters 5–9 |
| Kafka: The Definitive Guide | O'Reilly | 10h | We use Kafka daily — understand it deeply |
| Observability Engineering | O'Reilly | 8h | Metrics, traces, and structured logging |
| AWS Solutions Architect (Associate) | AWS Skill Builder | 20h | Required before joining on-call rotation |
| API Design Principles | Internal workshop | 4h | Monthly session with Platform Lead |

### Certification Goal

Target **AWS Solutions Architect – Associate** by end of IC2 tenure. Company reimburses exam fee ($300).

---

## Track 3: Senior Engineer (IC3–IC4)

**Goal:** Develop system design skills, technical leadership, and organizational influence.

### Required Learning

| Topic | Resource | Format |
|-------|----------|--------|
| Architecture trade-off analysis | *Fundamentals of Software Architecture* (Richards) | Book |
| Technical writing | Company technical writing workshop | Workshop (quarterly) |
| Production incident management | Shadow on-call for 1 full rotation | Shadowing |
| Team leadership | *The Manager's Path* Ch. 1–4 (even if staying IC) | Book |
| Rust foundations | Rustlings + *The Rust Programming Language* | Self-paced |

### Project Requirement

At IC3, each engineer leads one **cross-team technical initiative** per year:
- Write the design doc
- Get stakeholder alignment
- Coordinate with ≥ 2 other teams
- Deliver and document the outcome

This is the primary evidence for Staff Engineer promotion readiness.
